What is OpenEBS?

Why OpenEBS?

  • OpenEBS is designed in such a way that it is fully DevOps friendly
  • OpenEBS is not yet another scale-out storage system (YASSS)
  • OpenEBS is tightly integrated with Kubernetes and the part of Kubernetes-incubator Project
  • OpenEBS is most active Project which is written in Go

OpenEBS 0.4

Hyper-Converged storage for stateful applications on Kubernetes Platform
- A Developer friendly release.

Try Now

Containerized Storage for Containers

Your storage controller IS a container that follows your workloads around; very different than Yet Another Scale out Storage System (“!YASSS”).

Improves EBS & other cloud storage

Runs on top of EBS -to improve resiliency, to limit lock-in, and for better integration with Kubernetes.

Avoid cloud lock-in

When you write your data to OpenEBS - you write to an open source, highly flexible data management layer that allows you to manage your exposure to each cloud or data center.

Granular Control

Each workload and hence, each team working on dynamic container based workloads has their own storage, instead of being managed by central storage.

Written in GO

OpenEBS is the only broadly adopted open source block storage written mostly in Go.
